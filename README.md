## ⬡ Featured Projects

<details>
<summary><strong>⬡ RAG Application — Retrieval-Augmented Generation Pipeline</strong></summary>

<br/>

> A production-grade RAG system that ingests documents, builds a semantic vector store, and answers natural language queries with grounded, context-aware responses using Gemini's generative capabilities.

<div align="center">

| Attribute | Details |
|:---|:---|
| **Stack** | Python · LangChain · ChromaDB · Gemini API · FastAPI |
| **Scale** | Multi-document ingestion · chunked embedding pipeline · concurrent query handling |
| **Architecture** | Document loader → Text splitter → Embedding model → Vector store → Retriever → LLM chain |
| **Performance** | Sub-second retrieval from vector store · semantic similarity scoring per query |
| **Security** | API key scoping · input sanitization · no raw document exposure in responses |
| **Impact** | Demonstrates end-to-end LLM systems engineering beyond basic API wrappers |
| **Repository** | [![GitHub](https://img.shields.io/badge/CodeAritra%2FRag--App-6D28D9?style=flat-square&logo=github&logoColor=white)](https://github.com/CodeAritra/Rag-App) |

</div>

This project was built to demonstrate the full architecture of an AI-powered document Q&A system — from raw file ingestion through embedding generation, vector indexing, semantic retrieval, and finally LLM-based answer synthesis. ChromaDB was chosen for its lightweight self-hosted nature, making the system portable and infra-independent. The Gemini API handles generation with structured prompts that enforce answer grounding.

</details>
